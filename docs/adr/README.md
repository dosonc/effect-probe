# Architecture decision records

Architecture decision records capture decisions that materially constrain
EffectProbe's semantics, public interfaces, security boundary, or compatibility.
They explain why a decision was made and what remains deliberately unfrozen.

Statuses used in this directory are `Proposed`, `Accepted`, `Superseded`, and
`Rejected`. Accepted records are changed by adding a superseding ADR rather than by
rewriting their historical decision.
